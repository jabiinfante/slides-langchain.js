# Arquitecturas RAG

<v-clicks>

### 1. 2-Step RAG (Simple)
- Recuperación **siempre antes** de la generación
- Flujo lineal y predecible
- Ideal para: FAQs, chatbots de documentación
- ⚡ Alto control, baja flexibilidad

### 2. Agentic RAG
- Un **agente decide cuándo y cómo** recuperar información
- El LLM razona y accede a herramientas según necesite
- Ideal para: Asistentes de investigación con múltiples fuentes
- ⚡ Bajo control, alta flexibilidad

### 3. Hybrid RAG
- Combina ambos enfoques con **validación intermedia**
- Query enhancement + retrieval validation + answer validation
- Ideal para: Q&A de dominio específico con control de calidad
- ⚡ Control y flexibilidad medios

</v-clicks>
