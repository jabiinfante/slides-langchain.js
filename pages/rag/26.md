# Mejores prácticas RAG

<v-clicks>

### Chunking
- Tamaño típico: 500-1500 caracteres
- Overlap: 10-20% del tamaño del chunk
- Experimenta con diferentes estrategias

### Embeddings
- Usa el mismo modelo para indexar y buscar
- Considera modelos multiidioma si trabajas con español
- OpenAI `text-embedding-3-small` es un buen balance

### Búsqueda
- Recupera 3-5 documentos (más no siempre es mejor)
- Usa metadata para filtrar resultados
- Considera hybrid search (keyword + semántica)

### Prompts
- Instruye al LLM a basarse SOLO en el contexto
- Pide citas/referencias cuando sea posible
- Maneja casos donde no hay información relevante

</v-clicks>
